## 特記事項

### ディレクトリ名の末尾スペース

プロジェクトディレクトリ名は `logmee ` （末尾にスペースあり）。
パス指定時は必ず引用符で囲むこと:

```bash
cd "logmee "
```

### App.tsx の巨大さ

`App.tsx` は約200KBで全コンポーネント・状態管理・reducer を含む。
編集時は行番号を確認して対象箇所を特定すること。
新規コンポーネントもこのファイルに追加する（分割しない方針）。

### Tailwind CSS は CDN 版

npm パッケージではなく CDN スクリプトタグで読み込み。
カスタム設定は `index.html` 内の `<script>` タグで定義。
`tailwind.config.js` ファイルは存在しない。

### 外部ライブラリは importmap 経由

`index.html` 内の importmap で esm.sh CDN から読み込み。
npm パッケージとしてもインストール済み（型定義・ビルド用）。

### Supabase 接続情報

環境変数 `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` の設定が必須。
`.env.local` に設定すること（`.gitignore` 済み）。テンプレートは `.env.example` を参照。

> 2026-02-20: ハードコードされていた anon key / URL をソースコードから削除し、環境変数必須に変更。
> Vite dev server を `0.0.0.0` から `localhost` に制限。不要な GEMINI_API_KEY 設定を削除。

### localStorage バージョン

現在のストレージキー: `logmee_data_v16`
v12〜v15 からの自動マイグレーション対応済み。
スキーマ変更時はバージョンを上げてマイグレーション関数を更新すること。

## 現状の制約事項

1. **自動テストなし**: 型チェック（`npx tsc --noEmit`）と手動テストのみ
2. **単一ファイルアーキテクチャ**: App.tsx が巨大。IDE のパフォーマンスに影響する可能性
3. **CDN 依存**: Tailwind CSS, esm.sh のCDN可用性に依存
4. **オフライン限定対応**: localStorage に保存されるが、Service Worker なし
5. **単一ユーザー向け**: マルチテナントだが、チーム機能・共有機能なし
6. **ブラウザストレージ制限**: localStorage の容量制限（5-10MB）。大量のレポート保存時に注意
